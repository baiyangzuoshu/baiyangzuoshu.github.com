关卡随机掉落武将碎片算法

需求

  1. 关卡每两次随机掉落一次武将碎片
  2. 关卡每十次随机掉落一次武将碎片
  3. 每次最多掉落一次武将碎片

思考

  1. 数据的结构应该类似{“warID”:data}

  2. data的数据格式采用[0,1,0,1,0,1,0,1,2,1]

  3. 0表示未掉落，1表示掉落每两次，2表示每十次

  4. 每十次的掉落最终肯定是在0的时候进行判断；每两次的判断分为当前数组的长度%2取余数，即为偶数则随机，奇数的话，如果data[data.length-1]等于0，则必定会掉落，反之不掉落。

  5. 每次得到0，则每十次的ran+=20,然后取随机数进行比较，最终每十次掉落，则赋值为2

简读《李自成》

  七八月间开始读此书，我对传记人物的书籍颇为喜爱。之前看了《毛泽东传》，很受启发，心中感叹毛泽东可谓民族几千来数一数二的人物。为何会读《李自成》？它是第四届茅盾文学奖的作品，作者是姚雪垠先生。我看书都找50年以上的作品，这些作品经过时间的筛选留下来，值得一看！

  初看《李自成》讲述的是明朝末年时代背景，了解悲剧的崇祯皇帝、大义凌然的卢象升、骄狂失事的孙传庭等。崇祯时代的明朝之所以脆弱不堪，外表看来在于制度的老化，经济日益下滑。经济下滑，首先遭到冲击的就是老百姓。当老百姓的生计无法维持，就会官逼民反，随之出现农民暴动，进而消耗政府财力，如此循环直至朝代的更换。

  李自成，原名李弘基，儿时为地主牧羊。原为闯王高迎祥闯将，后高迎祥被孙传庭击败被俘而死，闯营公推李自成为新一代闯王。李自成在商洛山中的潜伏让我感慨，那段时光美好而让人憧憬，尽管物质匮乏，尽管官兵不断骚扰，种种困难都会被大家克服！那时候只要百姓看见李闯王，心中就有底，再苦再累都不怕，因为李闯王心系百姓啊，不忘百姓之艰辛！那时身边的那些人儿，慧英，慧珠，惠梅等高桂英手下的女将，各个不比男儿差！李过、李来亨、高一功、刘宗敏、田见秀、罗虎、张鼎等等这些大将，多好的男儿啊！

   当看见这些好男儿战死山海关，我难受不已。山海关之后，节节败退。最近心中仍然为此遗憾感到深深惋惜，这种大悲剧最是让我久久不能忘怀！悲剧给人以力量，因为疼痛会被记住。

初步了解hexo

环境搭建

node>=12.0

git

安装hexo

node install -g hexo-cli

hexo init <folder>

cd <folder>

npm install

配置ssh key

git config --global user.name "maomao"

git config --global user.email "baiyangzuo_shu@aliyun.com"

ssh-keygen -t rsa -C "baiyangzuo_shu@aliyun.com"

ssh -T git@github.com

部署GitHub

_config.yml配置deploy

npm install hexo-deployer-git --save

hexo clean && hexo generate && hexo deploy

基础命令

hexo init <folder>

hexo version

hexo clean

hexo generate

hexo publish

hexo deploy

hexo server

hexo list <type>

hexo config

hexo new "name"

  1. 将hexo误认为hero,在安装hexo-cli始终无法使用hexo init创建项目
  2. 添加ssh到GitHub时，邮件写错了。
  3. 测试GitHub访问页面，https://github.com/baiyangzuoshu/baiyangzuoshu.github.com.git，这个是不对的！
    https://baiyangzuoshu.github.io才是正确的链接
