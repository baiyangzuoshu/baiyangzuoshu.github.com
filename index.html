指针的输入和输出

指针这种技术很难理解，死记硬背是不行的，非得实操才能正真理解，即便你暂时理解了，没有间隔去复习还是会忘记！

int getMem(char** p1,int* len1,char** p2,int* len2){

char* tmp1=(char*)malloc(100);

strcpy(tmp1,"ssesjidjeidjijdiseji");

int tmp_len1=strlen(temp1);
p1=&tmp1;

len1=&tmp_len1;
char* tmp2=(char*)malloc(200);

strcpy(tmp2,"sfososkdoeo29ewjkfje");

int tmp2_len2=strlen(tmp2);

p2=&tmp2;

len2=&tmp2_len2;

return 0;

}

这是一段指针输入输出的代码，其中输出是有错误的。形参p1保存tmp1地址是无法作为输出，只有当p1=tmp1;当输入的一级指针保存tmp1在堆区申请的内存才有输出的效果！同理，len1=tmp_len1;也是一样的道理。

初步了解hexo

环境搭建

node>=12.0

git

安装hexo

node install -g hexo-cli

hexo init <folder>

cd <folder>

npm install

配置ssh key

git config --global user.name "maomao"

git config --global user.email "baiyangzuo_shu@aliyun.com"

ssh-keygen -t rsa -C "baiyangzuo_shu@aliyun.com"

ssh -T git@github.com

部署GitHub

_config.yml配置deploy

npm install hexo-deployer-git --save

hexo clean && hexo generate && hexo deploy

基础命令

hexo init <folder>

hexo version

hexo clean

hexo generate

hexo publish

hexo deploy

hexo server

hexo list <type>

hexo config

hexo new "name"

问题

  1. 将hexo误认为hero,在安装hexo-cli始终无法使用hexo init创建项目
  2. 添加ssh到GitHub时，邮件写错了。
  3. 测试GitHub访问页面，https://github.com/baiyangzuoshu/baiyangzuoshu.github.com.git，这个是不对的！
    https://baiyangzuoshu.github.io才是正确的链接
